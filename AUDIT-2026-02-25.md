## КРИТИЧЕСКИЕ ПРОБЛЕМЫ (P0)

### 1. Google API Key выставлен в `.env.example`
**Файл:** `.env.example`
**Строка:** `PAGESPEED_API_KEY=AIzaSy...`
Реальный ключ указан в примере. Нужно заменить на `YOUR_API_KEY_HERE` и убедиться что ключ в Railway secrets, а не в репозитории.

### 2. CORS полностью открыт
**Файл:** `app/main.py`
```python
allow_origins=["*"]
```
Любой сайт может делать AJAX запросы к API. Нужно ограничить origins для production.

### 3. Нет input validation от XSS/injection
**Файл:** `app/api/routes.py`
URL и текстовые поля принимаются как есть. Нет sanitization перед передачей в Playwright/subprocess.

### 4. LLM Worker не задеплоен
**Файл:** `railway.worker.toml`
Конфиг создан, но отдельный сервис на Railway не заведён. Без него LLM Crawler недоступен.

## СЕРЬЁЗНЫЕ ПРОБЛЕМЫ (P1)

### 5. `app/api/routes.py` — 6 310 строк в одном файле
Смешана логика парсинга robots.txt, валидации, экспорта, task management, всех 10 инструментов. Крайне сложно поддерживать. Нужна декомпозиция на модули.

### 6. `app/templates/task_progress.html` — 445K
Гигантский HTML-файл с inline JS. Загружается целиком при открытии любого результата. Нужна модуляризация по инструментам.

### 7. `app/tools/site_pro/adapter.py` — ~107K
Весь engine Site Audit Pro в одном файле. Сложно дебажить и расширять.

### 8. Нет rate limiting на основные endpoints
Rate limiting есть только в LLM Crawler. Основные инструменты (bot check, site audit, mobile) без защиты от злоупотреблений.

### 9. 4 падающих теста в LLM Crawler
**Файл:** `tests/test_llm_crawler_route.py`
Тесты flaky — иногда проходят, иногда нет. Причина: зависимость от Redis и worker heartbeat state.
